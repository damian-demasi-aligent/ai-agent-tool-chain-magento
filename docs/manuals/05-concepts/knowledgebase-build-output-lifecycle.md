# Build output lifecycle

The Vite build produces JS bundles and a CSS bundle in `web/js/` and `web/css/`. These are:

- **Not committed to git** — a pre-commit hook (`clean-build-output.sh`) removes them from staging
- **Generated by `yarn build`** in CI/CD and for local testing
- **Served by Magento** as static assets from `pub/static/`

During development, the dev server command (see CLAUDE.md → Commands) runs Vite's dev server which serves hot-reloading bundles over HTTPS. The `web/` directory is not used in dev mode.

**Common mistake:** Running `yarn build` and then `git add .` stages the build artifacts. The hook catches this for `git commit`, but if you're using `git add` independently, check `git status` before committing.
